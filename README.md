# UrbanFix Validation Backend

A **FastAPI** backend that validates user-submitted images (potholes / garbage) using **YOLOv8 ONNX** models. The API receives an image URL and category from the frontend, runs object detection, and updates the validation status in a **Supabase** database.

## Architecture Overview

```
Frontend                     Backend                         Supabase
   │                            │                               │
   │  POST /api/validate        │                               │
   │  {id, image_url, category} │                               │
   │ ─────────────────────────► │                               │
   │                            │  1. Fetch image via URL       │
   │                            │ ────────────────────────────► │
   │                            │  ◄──── image bytes ────────── │
   │                            │                               │
   │                            │  2. Run YOLO ONNX model       │
   │                            │     (pothole / garbage)       │
   │                            │                               │
   │                            │  3. UPDATE reports SET        │
   │                            │     validate = 'valid'/'invalid'
   │                            │ ────────────────────────────► │
   │                            │                               │
   │  ◄── {status, detections}  │                               │
   │                            │                               │
```

## Database Setup (Supabase)

### Step 1 — Create the table

Go to your Supabase dashboard → **SQL Editor** and run the following SQL:

```sql
-- Create the reports table (if it doesn't already exist)
CREATE TABLE IF NOT EXISTS reports (
    id          BIGINT GENERATED BY DEFAULT AS IDENTITY PRIMARY KEY,
    image_url   TEXT        NOT NULL,
    category    TEXT        NOT NULL CHECK (category IN ('pothole', 'garbage')),
    validate    TEXT        DEFAULT NULL,
    created_at  TIMESTAMPTZ DEFAULT now()
);
```

> **Note:** If you already have a `reports` table, just make sure it contains at least these columns:

| Column      | Type   | Description                                         |
| ----------- | ------ | --------------------------------------------------- |
| `id`        | BIGINT | Primary key – sent by the frontend                  |
| `image_url` | TEXT   | Public URL of the image in Supabase Storage          |
| `category`  | TEXT   | `"pothole"` or `"garbage"`                           |
| `validate`  | TEXT   | Updated by this backend: `"valid"` or `"invalid"`    |

### Step 2 — Add the `validate` column (if missing)

If your table already exists but doesn't have a `validate` column:

```sql
ALTER TABLE reports ADD COLUMN validate TEXT DEFAULT NULL;
```

### Step 3 — Enable Row Level Security (optional but recommended)

```sql
-- Allow the service role full access (backend uses service role key)
ALTER TABLE reports ENABLE ROW LEVEL SECURITY;

CREATE POLICY "Service role full access" ON reports
    FOR ALL
    USING (true)
    WITH CHECK (true);
```

### Step 4 — Verify your Storage bucket

Make sure:
1. Your images are uploaded to a **Supabase Storage bucket**.
2. The bucket is either **public**, or you're using signed URLs.
3. The `image_url` column stores the **full public URL** to the image.

Example public URL format:
```
https://<project-id>.supabase.co/storage/v1/object/public/<bucket-name>/<file-path>
```

## Environment Configuration

Copy the `.env` file and fill in your Supabase credentials:

```bash
# .env

# Your Supabase project URL (found in Project Settings → API)
SUPABASE_URL=https://your-project-id.supabase.co

# Your Supabase service role key (found in Project Settings → API → service_role)
# ⚠️  Use the SERVICE ROLE key, not the anon key — the backend needs write access.
SUPABASE_KEY=your-supabase-service-role-key

# Name of the database table to update
TABLE_NAME=reports

# YOLO detection thresholds (adjust if needed)
CONFIDENCE_THRESHOLD=0.5
IOU_THRESHOLD=0.45
```

| Variable                 | Description                                           | Default   |
| ------------------------ | ----------------------------------------------------- | --------- |
| `SUPABASE_URL`           | Your Supabase project URL                             | —         |
| `SUPABASE_KEY`           | Service role API key                                  | —         |
| `TABLE_NAME`             | Database table containing reports                     | `reports` |
| `CONFIDENCE_THRESHOLD`   | Minimum confidence for a detection to count           | `0.5`     |
| `IOU_THRESHOLD`          | IoU threshold for Non-Maximum Suppression             | `0.45`    |

## API Reference

### `GET /`

Health check endpoint.

**Response:**
```json
{ "status": "ok", "service": "UrbanFix Validation API" }
```

---

### `POST /api/validate`

Validate an image using YOLO object detection.

**Request Body:**
```json
{
    "id": 42,
    "image_url": "https://your-project-id.supabase.co/storage/v1/object/public/images/pothole_001.jpg",
    "category": "pothole"
}
```

| Field       | Type   | Required | Description                                    |
| ----------- | ------ | -------- | ---------------------------------------------- |
| `id`        | int    | ✅       | Row ID in the database to update               |
| `image_url` | string | ✅       | Full public URL of the image in Supabase Storage |
| `category`  | string | ✅       | `"pothole"` or `"garbage"`                      |

**Success Response (200):**
```json
{
    "id": 42,
    "status": "valid",
    "message": "Image validated as 'valid' with 3 detection(s).",
    "detections": 3
}
```

**Error Responses:**

| Status | Reason                                              |
| ------ | --------------------------------------------------- |
| 400    | Image URL is unreachable or file is not a valid image |
| 422    | Invalid request body (missing field, bad category)   |
| 500    | Model file missing or detection / database error     |

## Making Requests

### Using cURL

```bash
curl -X POST http://localhost:8000/api/validate \
  -H "Content-Type: application/json" \
  -d '{
    "id": 1,
    "image_url": "https://your-project-id.supabase.co/storage/v1/object/public/images/pothole_001.jpg",
    "category": "pothole"
  }'
```

### Using Python (httpx)

```python
import httpx

response = httpx.post(
    "http://localhost:8000/api/validate",
    json={
        "id": 1,
        "image_url": "https://your-project-id.supabase.co/storage/v1/object/public/images/pothole_001.jpg",
        "category": "pothole",
    },
)
print(response.json())
```

### Using JavaScript (fetch)

```javascript
const response = await fetch("http://localhost:8000/api/validate", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({
        id: 1,
        image_url: "https://your-project-id.supabase.co/storage/v1/object/public/images/pothole_001.jpg",
        category: "pothole",
    }),
});

const data = await response.json();
console.log(data);
```

## Troubleshooting

### "Model file not found" error
Make sure `pothole_best.onnx` and `garbage_best.onnx` are in the project root directory (same level as `main.py`).

### "Could not download image" error
- Verify the image URL is accessible in your browser.
- If your Supabase Storage bucket is **private**, make sure you're using signed URLs or switch the bucket to public.

### "Failed to update database" error
- Double-check your `SUPABASE_URL` and `SUPABASE_KEY` in `.env`.
- Make sure you're using the **service role key** (not the anon key).
- Verify the table name in `TABLE_NAME` matches your Supabase table.
- Ensure the `validate` column exists in the table.

### ONNX Runtime errors
- Make sure `onnxruntime` is installed: `pip install onnxruntime`
- If you have a GPU and want faster inference, install `onnxruntime-gpu` instead.

### Slow first request
The ONNX models are loaded into memory on the first request for each category. Subsequent requests reuse the cached model and will be significantly faster.
